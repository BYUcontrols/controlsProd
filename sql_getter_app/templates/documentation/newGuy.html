Where to Get Started

    I knew nothing about code when I started working on this app. You get to benefit from that because I started reading the codebase 
    and leaving comments about anything I learned. Hopefully that means that you can open any code file and see my comments to help
    you understand what's going on.
    
    However, I don't recommend reading code as a starting point because it is unproductive. Instead, pick a small task and try to complete it.
    For example, Matt told me to try duplicating a page as my first task. Solving this problem helped me understand a lot about how the app works.
    Don't get paralyzed. You will never know everything about code. You don't know what to look for anyway. Just get started on a simple problem
    and you will know what to search for after testing out a few things. Start by copying a line of code into Google and figuring out what it does.
    

Simple Project Ideas (in increasing difficulty)

    1) Change the color of a button. Then, change it back!
    2) Add some sample data to the ControlsTest database and make it appear in a table (this is simpler than you think).
    3) Duplicate a page. Be sure to add it to the menu so that you can access it! Once you have it working then remove it, of course.
    4) Make a small change, like the color of a button, and push that update to the test server. This is different than the dev site (look in 
       documentation for updating the test site).
    5) Consider asking Matt for a simple problem you can work on.

    If you do all 5 of these simple tasks then you will be well on your way to understanding the controls app!


Test Driven Development

    We are moving into the Test Driven Development model of software development.
    
    This is the process for TDD:
        1) Convert requirements into a test.
        2) Write the test method / function in code.
        3) Run the test. It should fail.
        4) Write code to pass the test.
        5) Run the test. It should pass.
        6) Every time you update your code you should run all tests and watch for failing tests.
    
    As you start writing code for the app, follow this process.



Pro Tips for New People

    1) Google anything you don't understand. Don't be afraid to copy an entire line of code into Google and search just to see what pops up.
    2) Use the dev environment to break things. Be sure to push changes to Git constantly so that you can revert back to old versions if you end up 
       breaking something beyond repair.
    3) Read all the other documentation pages on this site. If you need to know how to app works, look in the documentation first.
    4) Follow TDD from the beginning. It is much easier to maintain tests and code as you go along instead of building a backlog of tests and errors to 
       handle later.
    5) Act imperfectly. Progress is more important than perfection. Improve as you go along.
    6) Complete simple tasks instead of reading code (see above).
    7) Use comments to document your code as you go. Once again, its much easier to maintain updated documentation 
       if you do it as you go. It also helps you understand what you wrote when you come back to it further down the line.
